Match Arena Rules

Match and Validation Rules

1. Round-Robin Match: The match is played one round at a time between Tester and Coder .

2. Rules for testers:
  • Testers must use Playwright's CSS locator syntax to identify the "core elements" of the HTML code to be tested (as explained in the challenge description and comments);
  • Testers cannot use CSS syntax that identifies HTML tags via IDs, such as "#" or "id=";
  • Testers must respect the syntax and unambiguity constraints set by Playwright.
3. Rules for Coders:
  • Coders are authorized to tamper with the HTML code in such a way as to modify the relevant DOM tree and break the CSS selectors written in the test;
  • Coders may absolutely not tamper with the IDs of the "core elements" of the given HTML; furthermore, they are not authorized to duplicate them or reverse their original order;
  • To tamper with the tests, Coders may add or remove their own nodes to the DOM tree and/or change any other attribute to DOM elements, except for HTML tags.
4. The only difference compared to Playwright's standard behavior:
  • The Tester doesn't just need to make a selector work: selectors must necessarily identify and test the "core elements".
  • To the Coder will be only accepted submissions that break the test selectors with respect to the "core elements".

5. Watch out for the time! The challenge will continue until one of the two players runs out of time, declaring the other the winner.
